# 레퍼런스 정렬 — Align Reference (Stage 7: 정렬)

글을 시리즈·레퍼런스 글의 패턴에 맞춰 정렬합니다. 마지막 단계로 일관성 검증.

## 파이프라인 위치

```
/edit-writing → /tighten-structure → /clarify-writing → /refine-style → /smooth-flow → /verify-facts → /deep-review → ▶ /align-reference
```

## 인자
- `$ARGUMENTS`: 수정할 파일 경로

## 적용 axis (memory)
- 1순위: **의미론** (용어 일관)
- 2순위: **프로세스** (시리즈 규약)

## 점검 항목

### 1. 시리즈 용어·표기 일관성
- 시리즈(`-N`) 다른 파트와 핵심 용어·표기 동일 (예: "WebSocket" vs "웹소켓")
- 영어 약어 표기 (대소문자, 한국어 조사 결합)

### 2. 마무리 형식 (시리즈 규약)
- `## 마무리` 사용 (`## 정리` X)
- **관련 글** → **시리즈** 순서
- 본문 수정 후 마무리 수치·사실 동기화

### 3. 섹션 구조 (시리즈 규약)
- `##`/`###` 구조가 시리즈 다른 글과 호환
- 도입부·결론 위치·길이 일관성

### 4. 사전 지식 수준
- 시리즈 다른 파트가 전제하는 지식 수준과 일치
- 너무 깊거나 너무 얕지 않은지

### 5. 자산 표기
- 코드 블록 언어 명시
- 다이어그램 캡션·범례 형식 일관
- 시나리오 값 (IP, 포트, 호스트명) 시리즈 다른 글과 일치

## 출력 형식

```
[Stage 7: align-reference]
- 용어 일관: N건 통일
- 마무리 형식: 정규화 완료
- 섹션 구조: N건 조정
- 사전 지식 수준: ✓
- 자산 표기: N건 정렬
```

## 제약

- 정보 누락 0건
- 시리즈 규약과 충돌 시 시리즈 규약 우선
- 본문 사실 변경 X (verify-facts 단계가 담당)
