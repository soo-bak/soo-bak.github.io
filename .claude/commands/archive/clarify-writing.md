# 이해도 높이기 — Clarify Writing (Stage 2: 이해)

독자가 외부 지식 없이 글만으로 개념을 이해할 수 있도록 설명 기법을 점검하고 보완합니다.

## 파이프라인 위치

```
/edit-writing → /tighten-structure → ▶ /clarify-writing → /refine-style → /smooth-flow → /verify-facts → /deep-review → /align-reference
```

## 인자
- `$ARGUMENTS`: 수정할 파일 경로

## 적용 axis (memory)
- 1순위: **화용론** (전제 충족·정보 순서·구체 예시)

## 점검 항목

### 1. 설명 순서: "왜?" → "무엇?" → "어떻게?"
- 동기/문제 제시 후 정의·메커니즘
- 구체 예시 먼저, 추상 원리 도출은 그 다음 (화용론 2)

### 2. 빠진 전제 보강 (화용론 3)
- 독자가 알아야 할 전제는 발화 전에 제시
- "이해 안 됨" 사용자 피드백 = 빠진 전제 신호

### 3. 단계별 동사 서술 (화용론 2)
- 단계별 메커니즘을 라벨로 압축 X (예: "번들을 로드하고" → "번들 로드" 압축 금지)
- 초보 독자는 단계를 따라가며 이해

### 4. 결과·영향 명시 (화용론 9)
- 행동만 서술하고 결과 없으면 "그래서?" 발생
- 결과 연결 또는 "그래서 X가 됩니다" 보강

### 5. 지시어 일의성 (화용론 1)
- 모든 "이/그/저"가 직전 문장에서 일의적으로 해석되는지
- 후보 둘 이상이면 구체 명사로 교체

### 6. 메타 framing의 case 일치 (화용론 10)
- 주제 문장의 framing이 후속 case·예시와 의미적으로 일치
- 예: "그 비용이 다르다" framing → case들이 "추가 비용 변동"이면 "거기에 추가로 드는 비용"으로 framing 수정

### 7. 산문 우선
- 설명은 산문(문단)으로. 불릿은 병렬 비교·요약·체크리스트에만
