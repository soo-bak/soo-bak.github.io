# 심층 리뷰 — Deep Review (Stage 6: 독자 시뮬레이션)

파일 전체를 개념 단위로 자동 분할하고, 각 단위를 병렬 서브에이전트로 리뷰합니다. 4축 통합 점검.

## 파이프라인 위치

```
/edit-writing → /tighten-structure → /clarify-writing → /refine-style → /smooth-flow → /verify-facts → ▶ /deep-review → /align-reference
```

## 인자
- `$ARGUMENTS`: 수정할 파일 경로

## 적용 axis (memory)
- **4축 모두 통합 점검** (의미론·통사론·화용론·운율)
- 독자 시뮬레이션 관점에서 종합 검토

## 실행 절차

### Step 1: 개념 단위 분할
파일을 `##` 섹션 단위로 분할. 30줄 미만은 인접 섹션과 묶음.

### Step 2: 시리즈·자산 컨텍스트 수집
- 시리즈(`-N`) 글이면 같은 시리즈 다른 파트 용어·표기 추출
- 코드 블록·다이어그램·시나리오 값 인벤토리

### Step 3: 단위별 서브에이전트 리뷰

각 단위에 대해 `Agent(subagent_type="general-purpose")` 호출. 4축 점검:

**의미론** (`linguistic_semantics.md`):
- 기피 어휘·과장·단언
- 연결어미 ↔ 논리 관계 일치
- 용어 일관 (개념 = 한 표현)
- 주제 framing ↔ 후속 case 일치

**통사론** (`linguistic_syntax.md`):
- 자연 어순·주어·조사·명사화·피동·번역투
- 절 중첩·문장 호흡·단계 표지

**화용론** (`linguistic_pragmatics.md`):
- 지시어 일의성·정보 순서·전제 충족
- 메커니즘 반복·결과 명시·교과서 패턴

**운율** (`linguistic_prosody.md`):
- ~니다 분산·어휘 반복·문장 길이 교차
- 소리 내어 읽기 자연스러움

### Step 4: 독자 시뮬레이션
가상 초보자 독자 시점에서 3가지 점검:
1. **이해 궤적**: 어디서 멈추거나 되돌아 읽는가?
2. **개념 적층**: 새 개념이 등장 시 전제가 충분한가?
3. **흐름 단절**: 어느 문단·전환에서 끊기는가?

### Step 5: 통합 보고

```

- 의미론: N건 (구체 사례)
- 통사론: N건
- 화용론: N건
- 운율: N건

### 독자 시뮬레이션
- 이해 궤적 단절: [위치]
- 전제 부족: [위치]
- 흐름 단절: [위치]

### 적용 권장
[우선순위별 수정안]
```
