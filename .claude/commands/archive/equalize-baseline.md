# 기준선 보강 — Equalize Baseline (Stage 0)

밀도가 낮은 글의 기준선을 끌어올려 후속 파이프라인이 작동할 수 있게 합니다. **0→1**(없는 것을 만듦)에 집중. 톤·리듬·문장 다듬기는 후속 단계 담당.

## 파이프라인 위치

```
▶ /equalize-baseline → /edit-writing → /tighten-structure → /clarify-writing → /refine-style → /smooth-flow → /verify-facts → /deep-review → /align-reference
```

## 인자
- `$ARGUMENTS`: 수정할 파일 경로

## 적용 axis (memory)
- 1순위: **화용론** (전제 충족·정보 순서·섹션 도입 범위)
- 2순위: **프로세스** (Scope 선언·시리즈 규약)

## 점검 영역

### 영역 1: 설명 밀도 보강 (0→1만)
- 핵심 용어가 첫 등장 시 정의되어 있는가? 없으면 1문장 정의 추가
- 핵심 메커니즘이 한 번이라도 단계별로 서술되어 있는가? 없으면 추가
- 추상 단언만 있고 구체 예시·메커니즘이 없는 단락은 1개 예시 추가

### 영역 2: 도입부 맥락 연결
- 첫 문단이 직전 글(시리즈)·이번 주제와의 관계를 한 줄이라도 명시하는가?
- 도입문이 모든 하위 `###` 주제를 커버하는가? (화용론 11)

### 영역 3: 마무리 형식 정규화
- `## 마무리` 사용 (다른 표기 금지). **관련 글** → **시리즈** 순서
- 본문 수정 후 마무리 수치·사실이 동기화되어 있는가? (화용론 12)

### 영역 4: 시리즈 컨텍스트
- 시리즈(`-N` 접미사) 글이면 같은 시리즈의 용어·표기·사전 지식 수준 확인
- 충돌 시 시리즈 규약 우선

## 출력 형식

```
[Stage 0: equalize-baseline]
- 영역 1 (밀도): N건 추가
- 영역 2 (도입부): N건 보강
- 영역 3 (마무리): N건 정규화
- 영역 4 (시리즈): N건 동기화
```

## 제약

- 톤·리듬·문장 다듬기는 손대지 않음 (후속 단계)
- 추가만 허용, 기존 내용 삭제·재배치 금지
- 4축 점검은 후속 스테이지가 담당
