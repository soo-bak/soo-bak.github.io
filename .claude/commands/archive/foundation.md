# 기반 — Foundation (Stage 1/4)

글의 **뼈대**를 만들고 정리합니다. 기준선 보강(0→1) + 골격(구조·용어) + 구조 긴축(중복·재배치).

## 파이프라인 위치

```
▶ /foundation → /clarify → /polish → /verify
   기반           이해         다듬기      검증
```

## 인자
- `$ARGUMENTS`: 수정할 파일 경로

## 적용 axis (memory)
- 1순위: **통사론** (`linguistic_syntax.md`)
- 2순위: **의미론** (`linguistic_semantics.md`)
- 3순위: **화용론** (`linguistic_pragmatics.md`)
- 4순위: **프로세스** (`process_rules.md`)

## 점검 영역

### A. 기준선 보강 (0→1만)
- 핵심 용어 첫 등장 시 1줄 정의 — 없으면 추가
- 핵심 메커니즘이 단계별로 서술 — 없으면 추가
- 추상 단언만 있고 구체 예시·메커니즘 없는 단락 → 1개 예시 추가
- 도입부가 직전 글·이번 주제와의 관계 명시
- 도입문이 모든 하위 `###` 주제 커버 (화용론 11)
- `## 마무리` 형식 정규화 (다른 표기 X), **관련 글** → **시리즈** 순서

### B. 골격 (통사론·의미론)
- **용어 일관성**: 같은 개념 = 한 표현 (예: Visual Rebuild ≠ 메쉬 재생성 동시 사용 X)
- **기피 어휘** (의미론 1·4): 쪼개·가르·매우·훨씬·완전히·획기적·차원이 다른 등 → 매핑표
- **한국어 자연 어순**: 배경→주제, 조건→귀결, 원인→결과, 수식어→피수식어
- **절 중첩 3단계 이내**, 한 문장 쉼표 4개 이내
- **이중 피동·번역투** 회피 (~에 의해, ~되어질, ~로 하여금 등)
- **명사화 절제**: ~하는 것입니다, ~라는 점입니다 한 문단 2회+ X
- **단계 표지 명시**: 2단계+ 프로세스는 "먼저 ~ 그다음" (메타 announcement "두 단계로 이루어집니다" 단독 문장 X)

### C. 구조 긴축 (화용론)
- **메커니즘 반복 회피** (화용론 7): 같은 메커니즘 여러 문단 반복 X. 한 번 설명 후 결과만 서술
- **흐름 끊김 → 위치 이동** (화용론 6): A→B 사이 끼인 내용이 흐름 끊으면 압축 아닌 위치 이동
- **정보 순서** (화용론 2): 정의→예시→일반화→평가, 또는 배경→문제→해결. 추상→구체 X (구체 예시 먼저)
- **다이어그램·코드 후속** (화용론 8): 다이어그램 직후 첫 산문은 다이어그램 받기. 코드 출력 직후 산문은 같은 사실 재진술 X
- **결과 명시** (화용론 9): 행동만 서술하고 결과/영향 없으면 "그래서?" 발생 → 결과 연결
- **framing ↔ case 일치** (화용론 10·의미론 7): 주제 문장의 의미 framing이 후속 case·예시와 일치

## 출력 형식

```
[Stage 1: foundation]

A. 기준선 보강
- 용어 정의: N건 추가
- 메커니즘 서술: N건 추가
- 도입부·마무리: N건 정규화

B. 골격
- 용어 일관: N건 통일
- 기피 어휘: N건 교체
- 문장 구조: N건 재배치
- 단계 표지: N건

C. 구조 긴축
- 메커니즘 반복: N건 정리
- 흐름 끊김: N건 재배치
- 정보 순서: N건 수정
- 다이어그램 후속: N건 연결
- framing 일치: N건 점검
```

## 제약

- 이해 보강(전제·구체 예시 추가)은 다음 stage `/clarify` 담당
- 사실 검증은 마지막 stage `/verify` 담당
- 큰 재구성 필요 시 `/soobak-redesign` 호출
