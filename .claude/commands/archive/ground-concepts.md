# 개념 접지 — Ground Concepts

추상적 개념을 독자가 아는 일상 경험에 연결합니다. 기술 용어와 추상 설명을 독자의 경험에서 시작하여 이해할 수 있도록.

## 인자
- `$ARGUMENTS`: 수정할 파일 경로

## 핵심 원칙

**"독자가 모른다고 가정하되, 바보라고 가정하지 않는다."**

독자는 기술 용어를 모를 수 있지만 일상적 경험은 가지고 있다. 그 경험에서 출발하여 새 개념으로 연결.

## 적용 axis (memory)
- 1순위: **화용론** (구체→추상, 전제 충족, 메커니즘 단계 서술)

## 점검 항목

### 1. 추상 정의 → 구체 경험 연결
- 새 개념 첫 등장 시 일상 경험·이미 다룬 개념으로 비유·대응
- 예: "TCP 3-way handshake" → "약속 잡을 때 '내 말 들리지?' '응 들려' '좋아 시작하자'와 비슷한 절차"
- 단, 비유는 명료성 담보될 때만 (의미론 4: 과장 회피)

### 2. 단계별 동사 서술 (화용론 2)
- 메커니즘은 단계별로 풀어 씀 (라벨 압축 X)
- "번들 로드" → "번들을 메모리로 불러오고, 압축 해제하고, 객체로 직렬화하여 ..."

### 3. 구체 예시 먼저 (화용론 2)
- 추상 원리 도출 전 구체 예시 1개 제시
- "이런 경우를 생각해 보자: [구체]" → "결국 [원리]"

### 4. 빠진 전제 보강 (화용론 3)
- 독자가 알아야 할 전제는 발화 전 제시
- 시리즈 전 글에 정의된 개념은 1줄 요약 후 사용

### 5. 비유의 한계 명시
- 비유로 시작하더라도 한계점 명시 ("비유는 X까지만 통하고, 정확히는 Y")
- 비유가 잘못된 멘탈모델 유발 시 즉시 정확한 정의로 보강

### 6. 기술 용어 보존 (의미론 5)
- 정확 용어 ("L2", "필레이트", "텍셀")는 친숙어로 교체 X
- 정확 용어 + 1줄 설명이 정확 용어 삭제보다 우월

## 출력 형식

```
[Ground Concepts]
- 추상 → 구체 연결: N건 추가
- 단계 풀어 씀: N건
- 구체 예시 추가: N건
- 빠진 전제 보강: N건
- 비유 한계 명시: N건
```

## 제약

- 정확도 우선: 비유로 인한 부정확 < 정확한 기술 용어 + 1줄 설명
- 정보 누락 0건
- 메타 문장 ("이 글에서는 ~ 다룹니다") 허용 (사용자 선호)
