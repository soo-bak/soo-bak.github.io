# 섹션 사전 스캔 — Pre-Edit Scan

편집 요청 **전**에 대상 섹션의 모든 산문 품질 문제를 4축으로 한 번에 식별. 3-5회 반복 수정을 1회로 줄이는 것이 목적.

## 인자
- `$ARGUMENTS`: 파일 경로 + 선택적 섹션 (예: `MemoryManagement-3 ## 로딩 전략`)
- 인자가 없으면 사용자에게 대상 요청

## 실행 원칙

이 스킬은 **수정하지 않습니다**. 4축 점검으로 문제를 발견·분류·보고만 합니다.
보고 후 사용자가 "수정해줘"하면 모든 문제를 한 번의 Edit으로 해결.

## 실행 절차

### Step 1: 대상 파일·섹션 확정
- `$ARGUMENTS` 파싱. 시리즈명 패턴이면 `dev/{network,unity}/_posts/`에서 Glob
- 섹션 지정되면 해당 `##` 범위만, 미지정이면 전체 파일

### Step 2: 컨텍스트 수집
- 직전 섹션 마지막 2문장
- 직후 섹션 첫 2문장
- 다이어그램(SVG) 위치

### Step 3: 4축 점검

대상 텍스트를 Read 후, 4축 메모리 파일 기준으로 점검.

**의미론** (`linguistic_semantics.md`):
- 기피 어휘 (쪼개·가르·매우·훨씬 등)
- 연결어미 ↔ 논리 관계 일치 (~므로/지만/며/는데)
- 무조건 단언 ("절대/항상")
- 과장 부사·단언
- 같은 개념의 두 표현 혼용
- framing ↔ case 일치

**통사론** (`linguistic_syntax.md`):
- 어순·주어·격조사·명사화·피동·번역투
- 절 중첩·문장 호흡·단계 표지

**화용론** (`linguistic_pragmatics.md`):
- 지시어 일의성·정보 순서·전제 충족
- 메커니즘 반복·결과 명시·교과서 패턴
- 다이어그램·코드 후속·메타 framing

**운율** (`linguistic_prosody.md`):
- ~니다 분산·어휘 반복·문장 길이 교차
- 호흡·소리 내어 읽기

### Step 4: 대형 섹션(30줄+)은 서브에이전트

```
Agent(subagent_type="general-purpose", description="산문 품질 사전 스캔")
```

서브에이전트 프롬프트:
- 역할: 한국어 기술 블로그 산문 품질 전문 편집자
- 제약: **수정 X, Read만 사용**
- 입력: 파일 경로, 라인 범위, 직전·직후 컨텍스트
- 점검: 4축 메모리 파일 (`linguistic_*.md`)
- 출력: 4축별 위반 표

### Step 5: 보고 형식

```
## 사전 스캔 결과

### 의미론 (N건)
| # | 위치(line) | 문제 | 수정 방향 |
|---|---|---|---|
| 1 | L42 | "발생하지만 (대립 오용)" | "발생하며" 병렬 |
| 2 | L55 | "매우 큽니다" 과장 | "X에 비해 큽니다" 비교 |

### 통사론 (N건)
...

### 화용론 (N건)
...

### 운율 (N건)
...

**총 N건 탐지.**
```

위반이 없는 axis도 "✓ 위반 없음"으로 보고.

## 후속

사용자가 "수정해줘" 요청 시:
- 단순 패치 (1문장): 직접 Edit
- 2문장+ 또는 큰 재구성: `/soobak-redesign` 호출
