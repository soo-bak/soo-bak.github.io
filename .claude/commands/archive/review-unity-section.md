# 유니티 섹션 리뷰 — Review Unity Section

붙여넣은 텍스트의 원본 파일을 자동 탐지하고, **기술 리뷰 + 문체 수정**을 동시에 수행.

## 인자
- `$ARGUMENTS`: 붙여넣은 텍스트 (파일 경로 아님)

## 실행 원칙

**호출되면 항상 4단계 전체 빠짐없이 실행.** 섹션이 깔끔해 보이거나 변경이 적어 보여도 파이프라인 축약 X. 오케스트레이터는 충실히 실행하는 역할.

## 4단계 절차

### 1단계: 원본 파일 자동 탐지
1. 텍스트에서 **특징 구문 3개** 추출 (기술 용어 우선: "TBDR", "드로우콜", "프래그먼트 셰이더" 등). "## 마무리"·"관련 글" 같은 범용 문장은 회피.
2. 각 구문을 `dev/unity/_posts/` 디렉토리에서 `Grep`
3. **정확히 1개 파일**에서 모든 구문 매칭 → 확정. 0개·2개+ 매칭이면 사용자에게 경로 요청.
4. `Read`로 파일 전체 읽기, 붙여넣은 텍스트의 첫·끝 줄로 라인 범위 식별

### 2단계: 시리즈 컨텍스트 수집
- 파일명의 `[시리즈명-N]` 패턴 추출
- 같은 시리즈 다른 파트의 핵심 용어·표기·사전 지식 수준 확인

### 3단계: 4축 + 기술 점검

**언어학 4축** (`linguistic_*.md` 메모리):
- 의미론: 기피 어휘·연결어미·과장·용어 일관·framing
- 통사론: 어순·주어·조사·절 중첩·문장 호흡
- 화용론: 지시어·정보 순서·전제·메커니즘 반복·교과서 패턴
- 운율: ~니다 분산·어휘 반복·문장 길이 교차

**Unity 기술 점검**:
- Unity API·컴포넌트 명칭 정확성
- 렌더링·셰이더·메모리 관리 메커니즘 사실성
- 시리즈 다른 파트와의 기술 일관성
- 코드 예제 정확성

### 4단계: 일괄 수정 또는 보고
- 단순 수정(1-2 패치): 직접 Edit
- 2문장+ 또는 큰 재구성: `/soobak-redesign` 호출
- 보고 시 4축별·기술별 분류

## 출력 형식

```
## 유니티 섹션 리뷰

### 파일 확정: {경로} L{시작}-{끝}

### 4축 점검
- 의미론: N건
- 통사론: N건
- 화용론: N건
- 운율: N건

### Unity 기술
- API·명칭: N건
- 메커니즘: N건
- 시리즈 일관성: N건

### 적용 권장
[우선순위별 수정안]
```

## 제약

- 정보 누락 0건
- Unity 기술 정확성 우선 (가독성 > 정확성 X)
- 패치 한계: 같은 섹션 3+ Edit이면 `/soobak-redesign` 호출
