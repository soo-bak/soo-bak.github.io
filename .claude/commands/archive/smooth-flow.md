# 흐름 다듬기 — Smooth Flow (Stage 4: 흐름)

문장과 문단 사이를 막힘 없이 읽을 수 있도록 연결·전환·호흡을 다듬습니다.

## 파이프라인 위치

```
/edit-writing → /tighten-structure → /clarify-writing → /refine-style → ▶ /smooth-flow → /verify-facts → /deep-review → /align-reference
```

## 인자
- `$ARGUMENTS`: 수정할 파일 경로

## 적용 axis (memory)
- 1순위: **운율** (호흡·소리 내어 읽기)
- 2순위: **화용론** (전환·연결)
- 3순위: **의미론** (연결어미 ↔ 논리 관계 일치)

## 점검 항목

### 1. 연결어미 ↔ 논리 관계 일치 (의미론 2)
- 인과: ~므로 / ~기 때문에 / ~어서
- 대립: ~지만 / ~나
- 병렬: ~며 / ~고
- 배경+후속: ~는데
- 조건: ~면 / ~거든

**자주 발생하는 오용**: 병렬에 "반대로" 사용, 배경 제시에 "지만" 사용, 인과를 "~고"로 약화. 의미 관계와 일치하는지 확인.

### 2. 문단 간 전환 (화용론 6)
- 문단 시작 문장이 직전 문단의 결론을 받는가?
- A→B 사이 끼인 내용이 흐름 끊으면 위치 이동

### 3. 다이어그램·코드 후속 (화용론 8)
- 다이어그램 직후 첫 산문은 다이어그램 받기
- 코드 출력 직후 산문은 같은 사실 재진술 X

### 4. 호흡 분리 (운율 4)
- 한 문장이 두 번 읽어야 이해되면 분리
- 절 중첩 3단계+ 또는 쉼표 4개+ → 강제 분리

### 5. 짧은 문장 연결 (운율 3)
- 연속된 짧은 문장이 연결 없이 나열되면 목록처럼 읽힘
- 논리 관계(~하고/~하면/~도)로 병합하여 흐름 만들기

### 6. 소리 내어 읽기 (운율 5)
- Edit 후 첫 문단·인접 문장 소리 내어 자연스러운지 점검
