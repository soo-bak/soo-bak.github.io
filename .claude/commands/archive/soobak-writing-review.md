# 글쓰기 전체 리뷰 파이프라인 — soobak-writing-review

4단계 파이프라인을 순서대로 실행하여 글을 완전하게 수정합니다. 모든 단계는 메모리의 **언어학 4축**(`linguistic_*.md`) + **프로세스 규칙**(`process_rules.md`)을 적용.

```
/foundation → /clarify → /polish → /verify
   기반          이해         다듬기      검증
   (통사·의미·    (화용)      (운율·     (의미·4축)
    화용·프로세스)            의미·화용)
```

## 인자
- `$ARGUMENTS`: 수정할 파일 경로 (필수)

## 사전 분석

파이프라인 실행 **전** 다음 두 작업을 수행하고 모든 단계에 컨텍스트로 전달.

### 1. 시리즈 컨텍스트 수집
대상이 시리즈(파일명에 `-N` 접미사)인 경우, 같은 시리즈의 다른 파트에서 추출:
- 핵심 용어·표기법 (예: "WebSocket" vs "웹소켓")
- `##` 섹션 제목 목록
- 각 파트가 전제하는 사전 지식 수준

### 2. 파일 내 자산 인벤토리
- **코드 블록**: 위치, 언어, 줄 수, 주요 변수/함수 이름
- **다이어그램**: 위치, 엔티티 이름 목록
- **시나리오 값**: IP·포트·호스트명 등 구체 값

이 인벤토리는 이후 단계의 규칙·점검에서 참조.

## 실행 절차

`Skill` 도구로 아래 4개 커맨드를 **순서대로** 실행. 각 단계는 이전 단계의 수정 위에서 동작.

| Stage | 스킬 | 내용 | 주요 axis |
|------|------|------|------|
| 1 | `/foundation $ARGUMENTS` | 기준선 보강 + 골격 + 구조 긴축 | 통사·의미·화용·프로세스 |
| 2 | `/clarify $ARGUMENTS` | 빠진 전제·추상→구체·단계 서술·개념 접지 | 화용 |
| 3 | `/polish $ARGUMENTS` | 어말 분산·어휘·연결어·전환 | 운율·의미·화용 |
| 4 | `/verify $ARGUMENTS` | 사실·독자 시뮬레이션·시리즈 정렬 | 4축 통합 |

## 사용자 문체 규칙 (모든 단계 공통)

`MEMORY.md`의 **User Writing Preferences** 적용:
- 정확도 > 가독성 > 간결성. 정보 누락 금지
- 질문형 종결어미 X
- 정확 용어 보존 ("L2"→"이더넷" X)
- 교과서 패턴 X (~라는 것입니다, 중요한 점은, 살펴보았으니)
- 산문 우선 (불릿은 병렬 비교·요약·체크리스트)
- 초보자 기준 — 모든 전제 채우기
- 메타 문장 ("이 글에서는 ~ 다룹니다") 허용

## 단계 종료 후 점검

## 파이프라인 종료 후

수동 섹션별 리뷰는 사용자와 함께 진행. 자동 단계로 잡히지 않은 문제·문맥 의존 결정은 대화로 처리.

## 제약

- 정보 누락 0건 (process 7)
- 단계 간 충돌 시 4축 메모리의 원칙 우선
- 큰 재구성 필요 시 단계 진행 중 `/soobak-redesign` 호출 가능
- 4단계로도 해소되지 않는 문제는 4축 메모리 파일에 사례 추가 후 다음 회차 강화
