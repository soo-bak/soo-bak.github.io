# 구조 긴축 — Tighten Structure (Stage 1.5)

중복 감사 + 배치 최적화. 같은 정보가 여러 곳에 반복되거나 잘못된 위치에 있는 것을 정리합니다.

## 파이프라인 위치

```
/edit-writing → ▶ /tighten-structure → /clarify-writing → /refine-style → /smooth-flow → /verify-facts → /deep-review → /align-reference
```

## 인자
- `$ARGUMENTS`: 수정할 파일 경로

## 적용 axis (memory)
- 1순위: **화용론** (메커니즘 반복·흐름 끊김·정보 순서)
- 2순위: **통사론** (절 중첩·문장 호흡)

## 점검 항목

### 1. 메커니즘 반복 (화용론 7)
- 같은 메커니즘을 여러 문단에서 구체 이름만 바꿔 반복하지 않았는가?
- 한 번 설명 후 결과만 서술하도록 정리

### 2. 흐름 끊김 → 위치 이동 (화용론 6)
- A→B 사이에 끼인 내용이 흐름을 끊으면 압축이 아니라 위치 이동
- 적절한 섹션으로 이동, 또는 별도 문단으로 분리

### 3. 정보 순서 (화용론 2)
- 정의→예시→일반화→평가 순서
- 추상→구체 X (구체 예시 먼저)
- 단계별 메커니즘은 동사 서술 (라벨 압축 X)

### 4. 다이어그램·코드 후속 (화용론 8)
- 다이어그램 직후 첫 산문은 다이어그램 받기
- 코드 출력 직후 산문은 같은 사실 재진술 X

### 5. 결과 명시 (화용론 9)
- "~합니다"로 끝나는 행동 서술이 결과/영향 없이 끝나면 "그래서?" 발생 → 결과 연결

### 6. 같은 메커니즘 framing 일치 (화용론 10)
- 주제 문장의 framing이 후속 case·예시와 의미적으로 일치하는가?
