# 글쓰기 수정 스킬

주어진 글을 아래 가이드라인에 따라 수정합니다.

## 인자
- `$ARGUMENTS`: 수정할 파일 경로 (선택사항). 없으면 사용자에게 파일 또는 텍스트를 요청합니다.

## 수정 원칙

### 1. 문장의 자연스러움
- 교과서적 표현 피하기: "~라는 것입니다" → 직접 서술
- "중요한 점은 ~입니다" → 바로 내용 서술
- 소리 내어 읽었을 때 자연스러워야 함

### 2. 직관적인 이해
- 한 문장에 하나의 개념
- 정의 → 예시 → 의미/결과 순서
- 이유 없이 결론만 제시하지 않기
- 독자가 추론해야 하는 논리적 간극 제거

### 3. 간결하지만 완전한 설명
- 불필요한 반복 제거
- 필요한 정보는 반드시 포함
- "없어도 의미 전달에 지장 없는 단어" 제거

### 4. 문장 구조
- 짧은 문장 선호 (2줄 이내)
- 주어-서술어 관계 명확
- 대명사보다 명확한 지시어

### 5. 객관적 문체
- 주관적 표현 삼가: "놀라운", "중요한", "매우"
- 비유 삼가: "마치 ~처럼"
- 강조는 구조로 (별도 문단, 굵은 글씨)

### 6. 문단 구조
- 하나의 문단에 하나의 주제
- "첫째", "둘째"로 논리 구분
- 앞뒤 문단 연결 명시

### 7. 단어 선택
- 같은 개념에 같은 용어 일관 사용
- 새 용어는 첫 등장 시 정의
- 자연스러운 조합: 파장-길다/짧다, 주파수-높다/낮다

### 8. 글의 흐름과 맥락
- 앞에서 설명한 개념이 뒤에서 활용
- 트레이드오프 관계 명시
- "앞서 살펴본 것처럼" 등으로 연결

### 9. 정확성
- 인과 관계가 논리적으로 맞는지 확인
- 불확실하면 조사 후 작성
- 상관관계를 인과관계로 서술하지 않기

## 작업 프로세스

1. **파일 읽기**: `$ARGUMENTS`가 있으면 해당 파일 읽기
2. **분석**: 각 원칙에 맞지 않는 부분 식별
3. **수정**: 가이드라인 적용하여 수정
4. **검토**: 원래 의미 훼손 여부 확인
5. **출력**: 수정된 글 + 주요 변경 사항 요약
