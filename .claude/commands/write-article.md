# 글 작성 — Write Article

주어진 주제에 대해 **언어학 4축을 작성 시점부터 적용**하여, 추가 수정이 거의 필요 없는 완전한 글을 작성합니다. 단독 글·시리즈·개념 글·실전 글 모두 지원.

## 핵심 원칙

**"고친다"가 아니라 "처음부터 잘 쓴다."**

작성 후 `/foundation`·`/clarify`·`/polish`·`/verify` 단계가 **거의 일거리가 없을 정도로** 4축이 이미 만족된 글을 생성하는 것이 목표.

## 인자
- `$ARGUMENTS`: 주제 + (선택) 글 종류·시리즈 여부·분량·톤
- 인자가 부족하면 사용자에게 한 번에 모두 질문 (글 종류·시리즈·분량·핵심 다룰 개념 등)

## 적용 axis (memory) — 작성 시점부터 모두 적용
- **의미론** (`linguistic_semantics.md`): 어휘·연결어·논리·과장·용어 일관·framing
- **통사론** (`linguistic_syntax.md`): 어순·주어·조사·명사화·피동·절 중첩
- **화용론** (`linguistic_pragmatics.md`): 정보 순서·전제·지시어·메커니즘·교과서 회피
- **운율** (`linguistic_prosody.md`): 어말 분산·어휘 반복·문장 길이
- **프로세스** (`process_rules.md`): scope 선언·정보 보존·시리즈 규약

## 작성 절차

### Step 1: 주제 분석 + Scope 선언

응답에 다음 명시:
- 핵심 개념 → 1줄 정의
- 독자 사전 지식 가정 (초보자 기준)
- **(a) 주제 범위**: 다루는 것 / 다루지 않는 것
- **(b) 외부 참조 범위**: 인용할 외부 글이 있다면 범위 차이 점검
- **(c) 개념 층위**: 중의적 용어가 있다면 본 글에서 사용할 층위 고정

### Step 2: 글 종류·구조 결정

**단독 글 vs 시리즈**:
- 단독: 한 글로 완결되는 주제
- 시리즈: 2~3파트로 분할 가능한 주제. 파트 분할 기준 명시 (예: 개념→구현→최적화)

**시리즈인 경우 컨텍스트 수집**:
- 같은 시리즈명 패턴(`-N`) 다른 글의 용어·표기·사전 지식 수준 추출
- 새 글이 이전 파트와 모순 없도록 정합성 점검

### Step 3: 골격 설계

- 도입부: 직전 글·이번 주제와의 관계 + 본 글 다룰 범위
- `##` 섹션 목록 + 각 1줄 요지
- 결론·마무리: `## 마무리` + **관련 글** + (시리즈인 경우) **시리즈**

각 섹션 골격에서 화용론 적용:
- 정보 순서: 정의→예시→일반화→평가 또는 배경→문제→해결
- 도입문이 모든 하위 `###` 주제 커버하도록 설계 (화용론 11)

### Step 4: 본문 작성 — 4축 동시 적용 루프

각 문단을 작성하며 4축을 **사후 점검이 아닌 작성 시점에 적용**:

**작성 전 (문단 단위)**:
- 이 문단이 답할 독자의 즉각적 질문 한 줄
- 정보 순서가 추상→구체 아닌 구체→추상 (화용론 2)
- 빠진 전제 미리 보강 (화용론 3)

**작성 중**:
- 의미론: 정확 용어 선정, 연결어미가 실제 논리 관계와 일치, 단언/과장 회피
- 통사론: 자연 어순, 절 중첩 3단계 이내, 단계 표지 명시
- 운율: 같은 어간 ~니다 연속 3회+ 회피, 같은 단어 한 문단 3회+ 회피, 짧고 긴 문장 교차

**작성 후 (문단 단위, 다음 문단으로 넘어가기 전)**:
- 직전 문단 + 본 문단 소리 내어 자연 흐름 확인
- 4축 위반 발견 시 그 자리에서 즉시 재작성 (사용자 피드백 기다리지 않기)
- 위반 없으면 다음 문단으로

### Step 5: 자산 추가

**다이어그램** (개념·메커니즘·구조):
- SVG, `viewBox`, `currentColor` (다크모드 호환)
- `font-family="sans-serif"` 명시
- 다이어그램의 "→" 결론 라인은 **중복이 아님** — 보존

**코드 블록**:
- 언어 명시 (` ```ts `, ` ```cs ` 등)
- 핵심 변수·함수 이름은 본문 용어와 일치

**표**:
- 병렬 비교·요약·체크리스트에만 사용
- 일반 설명은 산문 우선

### Step 6: 자체 4축 점검 + 패치 한계 자가 진단

작성 완료 후 응답에 4축 점검 결과 명시:

```
재구성 후 점검:
- 의미론: ✓ / ⚠️ [구체 사항]
- 통사론: ✓ / ⚠️
- 화용론: ✓ / ⚠️
- 운율: ✓ / ⚠️
정보 보존: ✓ (생성 글이므로 보존 대상 없음)
```

위반 발견 시 즉시 자가 재작성. 같은 문단을 2회 이상 자가 수정해야 한다면 그 문단을 **처음부터** 다시 작성 (process 3).

### Step 7: 사용자 검수 안내

작성 완료 후 사용자에게 보고:
- 작성된 파일 경로
- Scope 선언 요약
- 4축 점검 결과
- "추가 수정이 필요하면 어느 부분을 어떤 관점으로 수정하면 좋을지 알려주세요"

## 출력 형식

```
## 글 작성 완료

### 작성된 파일
- {경로 1}
- {경로 2 (시리즈 일 때)}
- ...

### Scope 선언
- (a) 주제 범위: ...
- (b) 외부 참조: ...
- (c) 개념 층위: ...

### 글 종류·구조
- {단독 / 시리즈 N파트}
- 섹션 구성: {요약}

### 4축 자체 점검
- 의미론·통사론·화용론·운율: 모두 ✓ / ⚠️ N건 (구체)
- 정보 보존: ✓
- 자가 재작성 횟수: N회

### 후속
추가 수정이 필요하면 `/soobak-redesign`으로 단일 블록 재작성, 또는 `/foundation`/`/clarify`/`/polish`/`/verify` 단계 호출.
```

## 사용자 문체 규칙 (반드시 적용)

- 정확도 > 가독성 > 간결성. 정보 누락 금지
- 질문형 종결어미 X
- 정확 용어 보존 ("L2"→"이더넷" X. "필레이트/텍셀/드로우콜" 같은 정확 용어 절대 누락 X)
- 교과서 패턴 X (~라는 것입니다, 중요한 점은, 살펴보았으니, 다음과 같이)
- 산문 우선 (불릿은 병렬 비교·요약·체크리스트만)
- 초보자 기준 — 모든 전제 채우기
- 메타 문장 ("이 글에서는 ~ 다룹니다") 허용
- "별도 글에서 다루는 것이 적합합니다" 절대 X — 맥락에 등장한 주제는 그 자리에서 완전 설명

## 제약

- 정확도 절대 우선 — 비유로 인한 부정확보다 정확한 기술 용어 + 1줄 설명이 우월
- 정보 누락 0건 (process 7)
- 1차 작성에서 4축 위반 발견 시 그 자리에서 자가 재작성 (사용자 피드백 기다리지 않기)
- 큰 재작성이 필요하면 `/soobak-redesign` 호출 가능
- 시리즈 규약과 충돌 시 시리즈 규약 우선
- "~별도 글에서 다룹니다", "여기서는 다루지 않습니다" 같은 회피 금지 — 본 글 범위 안의 주제는 모두 완전히 설명
