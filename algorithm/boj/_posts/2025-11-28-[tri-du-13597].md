## 문제 링크
[13597번 - Tri-du](https://www.acmicpc.net/problem/13597)

## 설명

카드 두 장의 값 A, B (1 ≤ A, B ≤ 13)가 주어지는 상황에서, 승리 확률을 최대화하기 위해 세 번째 카드로 선택할 값을 출력하는 문제입니다.

<br>

## 접근법

Tri-du 게임에서는 동일한 값이 많을수록 유리합니다. 트리오(같은 값 3개)가 듀오(같은 값 2개)보다 강하고, 듀오가 단일 카드보다 강합니다.

따라서 세 번째 카드로 이미 가진 두 카드 중 하나와 같은 값을 선택하여 듀오 이상을 만드는 것이 유리합니다.

<br>
경우를 나누어 보면:
- A = B인 경우: 같은 값을 선택하면 트리오가 됩니다.
- A ≠ B인 경우: 둘 중 더 큰 값을 선택하면 듀오를 만들면서 높은 값을 유지합니다.

<br>
두 경우 모두 `max(A, B)`를 선택하는 것이 최적입니다.

A = B일 때는 `max(A, B) = A = B`이므로 트리오가 되고, A ≠ B일 때는 더 큰 값으로 듀오를 만듭니다.

<br>

## Code

### C#

```csharp
using System;

namespace Solution {
  class Program {
    static void Main(string[] args) {
      var parts = Console.ReadLine()!.Split();
      var a = int.Parse(parts[0]);
      var b = int.Parse(parts[1]);

      Console.WriteLine(Math.Max(a, b));
    }
  }
}
```

### C++

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
  ios::sync_with_stdio(false);
  cin.tie(nullptr);

  int a, b; cin >> a >> b;

  cout << max(a, b) << "\n";

  return 0;
}
```
