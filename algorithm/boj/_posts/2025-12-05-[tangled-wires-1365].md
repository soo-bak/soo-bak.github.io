## 문제 링크
[1365번 - 꼬인 전깃줄](https://www.acmicpc.net/problem/1365)

## 설명

왼쪽 전봇대 i와 오른쪽 전봇대 a[i]가 연결되어 있을 때, 교차하는 전선이 없도록 최소한의 전선을 잘라야 하는 문제입니다.

교차를 없애려면 오른쪽 번호가 증가하는 순서로만 남겨야 합니다. 따라서 오른쪽 번호 배열의 가장 긴 증가하는 부분 수열(LIS) 길이만큼 남기고 나머지를 제거하면 됩니다.

<br>

## 접근법

이분 탐색 기반 LIS 알고리즘을 활용하여 가장 긴 증가하는 부분 수열의 길이를 구합니다.

<br>
먼저 lis 배열을 준비합니다. lis[i]는 길이가 i+1인 증가 부분 수열 중 가장 작은 마지막 값을 저장합니다.

다음으로 각 값 x에 대해 이분 탐색으로 lis 배열에서 x 이상인 첫 위치를 찾습니다. 해당 위치에 x를 넣고, 만약 배열 끝에 추가되는 경우라면 LIS 길이를 1 증가시킵니다.

이후 모든 원소를 처리하면 최종 LIS 길이 len을 얻습니다. 최소 제거 전선 수는 N - len이 됩니다.

<br>
시간 복잡도는 O(N log N)입니다.

<br>

## Code

### C#

```csharp
using System;

namespace Solution {
  class Program {
    static void Main(string[] args) {
      var n = int.Parse(Console.ReadLine()!);
      var arr = Array.ConvertAll(Console.ReadLine()!.Split(), int.Parse);

      var lis = new int[n];
      var len = 0;
      for (var i = 0; i < n; i++) {
        var x = arr[i];
        var idx = Array.BinarySearch(lis, 0, len, x);
        if (idx < 0)
          idx = ~idx;
        lis[idx] = x;
        if (idx == len)
          len++;
      }

      Console.WriteLine(n - len);
    }
  }
}
```

### C++

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef vector<int> vi;

int main() {
  ios::sync_with_stdio(false);
  cin.tie(nullptr);

  int n; cin >> n;
  vi a(n);
  for (int i = 0; i < n; i++)
    cin >> a[i];

  vi lis(n);
  int len = 0;
  for (int x : a) {
    auto it = lower_bound(lis.begin(), lis.begin() + len, x);
    *it = x;
    if (it == lis.begin() + len)
      len++;
  }

  cout << n - len << "\n";

  return 0;
}
```
