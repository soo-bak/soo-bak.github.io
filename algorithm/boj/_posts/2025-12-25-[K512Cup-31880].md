## 문제 링크
[31880번 - K512컵 개최!](https://www.acmicpc.net/problem/31880)

## 설명
덧셈 주문서와 곱셈 주문서를 모두 사용해 만들 수 있는 최대 행운 수치를 구하는 문제입니다.

<br>

## 접근법
곱셈 값이 0이면 먼저 적용해도 손해가 없고, 곱셈 값이 1 이상이면 덧셈을 먼저 하는 것이 항상 이득입니다.  
0인 곱셈은 먼저 처리해도 손해가 없고, 1 이상인 곱셈은 덧셈을 모두 한 뒤 적용하는 것이 항상 유리합니다.  
따라서 덧셈의 총합에 0이 아닌 곱셈 값들의 곱을 곱한 값이 최댓값입니다.

- - -

## Code

### C#
```csharp
using System;

class Program {
  static void Main() {
    var parts = Console.In.ReadToEnd().Split();
    var idx = 0;
    var n = int.Parse(parts[idx++]);
    var m = int.Parse(parts[idx++]);

    long sum = 0;
    for (var i = 0; i < n; i++)
      sum += int.Parse(parts[idx++]);

    long prod = 1;
    for (var i = 0; i < m; i++) {
      var b = int.Parse(parts[idx++]);
      if (b > 0) prod *= b;
    }

    Console.WriteLine(sum * prod);
  }
}
```

### C++
```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

int main() {
  ios::sync_with_stdio(false);
  cin.tie(nullptr);

  int n, m; cin >> n >> m;
  ll sum = 0;
  for (int i = 0; i < n; i++) {
    int a; cin >> a;
    sum += a;
  }

  ll prod = 1;
  for (int i = 0; i < m; i++) {
    int b; cin >> b;
    if (b > 0) prod *= b;
  }

  cout << sum * prod << "\n";

  return 0;
}
```
