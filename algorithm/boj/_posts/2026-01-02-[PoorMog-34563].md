## 풀이 아이디어
- 가로선과 세로선이 수직이 되도록 각도를 고정하려면, 한 직선이 이미 각도가 정해진 세로선과 한 번이라도 교차하면 그 가로선의 각도도 고정된다. 세로선도 마찬가지다.
- 따라서 기준이 될 가로선 하나를 잡고, 이 가로선과 **모든** 세로선을 커넥터로 고정하면 모든 세로선의 각도가 결정된다. 필요한 커넥터 수는 `M`.
- 이제 남은 가로선들은 각자 이미 고정된 세로선 중 아무거나 하나와만 고정하면 각도가 결정된다. 필요한 커넥터 수는 `N-1`.
- 합치면 `M + (N-1) = N + M - 1`. 기준을 세로선으로 잡아도 동일한 식이 나온다.
- 이는 최적이다. 각 가로선이 최소 한 번, 각 세로선이 최소 한 번은 고정돼야 하고(서로 수직을 이루기 위해), 이 조건을 만족하는 최소 간선 수가 바로 `N + M - 1`이다(트리 형태).

## 복잡도
- 테스트 케이스당 O(1) 계산.
- 입력 크기 합 제약(∑N·M ≤ 10⁶) 내에서 충분하다.

## 코드
### C#
```csharp
using System;

class Program {
  static void Main() {
    int T = int.Parse(Console.ReadLine()!);
    for (int tc = 0; tc < T; tc++) {
      var parts = Console.ReadLine()!.Split();
      long N = long.Parse(parts[0]);
      long M = long.Parse(parts[1]);
      Console.WriteLine(N + M - 1);
    }
  }
}
```

### C++
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
  ios::sync_with_stdio(false);
  cin.tie(nullptr);

  int T; 
  if (!(cin >> T)) return 0;
  while (T--) {
    long long N, M; cin >> N >> M;
    cout << (N + M - 1) << "\n";
  }
  return 0;
}
```
